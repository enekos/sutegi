⚒️ the forge · zero-dependency · agent-native

Laravel for Rustbuilt from std up.

sutegi (Basque: forge) is a batteries-included web framework with zero third-party dependencies. The HTTP/1.1 server, JSON, router, ORM, and LLM tool layer are all hand-built on std. No tokio, no serde — a tiny binary, and an AI agent as a first-class user.

// what is sutegi

Three goals usually in tension, held at once.

Batteries-included ergonomics, a minimal binary, and an interface built for autonomous agents — reconciled by one decision: build every component from the standard library, with no async runtime.

📦

From the ground up

Original std-only code you can read in an afternoon. Hand-written HTTP/1.1 parser, JSON codec, router, query builder.

🪶

Minimum binary

No async runtime, size-optimized release profile. The core "hello" service is ~362 KB; full SQLite build ~1.3 MB.

🤖

Agent-native

Routes, models, and tools are introspectable as JSON at runtime; tool-calling is a first-class concept, not a bolt-on.

// features

A real framework, not a toy.

Each pillar is an opt-in compile-time feature, so you ship only what you use.

🧭

Routing & middleware

Path params, route groups, before/after middleware, typed extractors — the request→handler→response spine.

🗄️

ORM & query builder

Typed schema, parameterized SELECT/UPDATE/DELETE builders, migrations, transactions. Optional bundled SQLite.

🧬

#[derive(Model)]

Typed models hydrate from rows and serialize to JSON. Build-time-only macro deps never reach your binary.

✅

Validation

Laravel-style rule sets and a JSON Schema validator, with structured per-field errors. Reused for AI tool args.

📡

Streaming & SSE

Stream bytes or Server-Sent Events with natural backpressure — the transport for live LLM tokens.

⚙️

Background jobs

An in-process queue with workers, retries, delayed dispatch, and introspectable stats.

🔭

Introspection

GET /__introspect returns the whole app surface as JSON — routes, models, tools — for humans and agents.

🧱

Hexagonal toolkit

AppError, UseCase ports, and adapter glue for clean, testable ports-and-adapters apps.

☸️

Pod-ready

Health/readiness/metrics endpoints, graceful SIGTERM draining, and a Sail-style Docker/k8s harness.

// the agent contract

An app an LLM can drive — no SDK, no source access.

Every sutegi app is self-describing over plain JSON/HTTP. The loop is discover → manifest → invoke.

GET /__introspectFull surface: routes (with docs), data models, and tools — deterministic JSON.
GET /__toolsAnthropic-style {name, description, input_schema, streaming} tool manifest.
POST /__tools/:nameInvoke a tool with a JSON body; arguments validated against its schema (precise 422 on failure).
POST /__tools/:name/streamInvoke a streaming tool; response is Server-Sent Events (live tokens).
// architecture

Opinionated hexagonal, when you need structure.

The hex toolkit nudges you toward ports & adapters: the dependency rule keeps your domain free of HTTP/SQL and your business logic testable without a server. One use case, many transports. 

   inbound adapters                       outbound adapters
 (HTTP handlers, AI tools)              (SQLite repo, HTTP client)
          │  call                            ▲  implement
          ▼                                  │
  ┌───────────────┐     depends on    ┌──────────────┐
  │  application  │ ─────────────────▶│    ports     │
  │  (use cases)  │                   │   (traits)   │
  └───────┬───────┘                   └──────────────┘
          │ uses
          ▼
  ┌───────────────┐
  │    domain     │   pure: entities + rules, no framework
  └───────────────┘
